# Space Complexity Analysis
## Core Concepts
- **Auxiliary space** vs total space
- **In-place algorithms** (O(1) extra space)
- **Space-time tradeoffs**
- Stack space in recursion
- Memory layout considerations
## Complexity Classes
- O(1) - Constant extra space
- O(log n) - Recursive call stack (binary search, divide-conquer)
- O(n) - Linear space (hash tables, frequency arrays)
- O(n??) - 2D arrays, matrices
## Python Specifics
- List vs generator memory usage
- String immutability implications
- Dict/set overhead (~3x element size)
- sys.getsizeof() for measurement
